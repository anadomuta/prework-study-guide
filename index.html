HTML

  • Structure of HTML:
    • !DOCTYPE html declaration: tells the browser what document type it should expect when it is opened
    • html: represents the root, or start, of our HTML document.
    • lang="en" attribute: tells the browser the language for the webpage
    • head: information about the webpage and helps the browser to render the page correctly.
    • body: visible content shown to the user.
    • metadata: data about our webpage that the browser uses to effectively serve our webpage to users
  • Semantic HTML elements: name describes its content and the role it plays on the webpage.
    Examples:
    1. body
  • Non-semantic elements:
    1. div
    2. span
  • It is best practice to use semantic elements whenever possible, especially when the content is visible to users.
  • HTML elements can be nested inside other elements. The enclosed element is the child of the enclosing parent element.

CSS

  • 3 ways to style a webpage using CSS:
    1. Inline CSS: style element directly in the HTML
    2. Internal CSS
    3. External CSS: most common, allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • selector: element or attributes to which the rules, or declarations, will apply
  • margin: how much space we want around the outside of an element
  • padding: adds space around the content inside an element
  • * selector: apply a rule to all the elements visible on the page

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git diff --check: identifies possible whitespace errors and lists them for you
  • git pull origin main: ensures that you have the latest version of the project on your local computer before you start to work.
  • GitHub Workflow:
    1. Check which branch you're currently on: git status
    2. Create a new branch off of the most recent version of the main branch: git checkout -b feature-name
    3. Work locally off of this new branch in Github Desktop
      • Best Practices:
        • Make sure you are working on the current repository
        • Make sure that you are correct branch
        • Make sure you have the most recent version of that branch
    4. Add or make content change in markdown files
    5. Commit changes to this branch
